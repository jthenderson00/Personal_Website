How I work

Useful technology beats impressive-looking technology.

My strongest work lives where support, systems administration, and automation overlap. I’m comfortable moving from a user-facing issue to logs, networking, storage, virtualization, or code when that is what the problem actually requires.

Troubleshoot from evidence

I inspect symptoms, logs, dependencies, and recent changes before reaching for a dramatic fix. The goal is a defensible root cause, not a lucky reboot.

Automate the boring parts

Once a process is understood, I turn repeat work into safe scripts, checks, reports, or scheduled workflows with clear failure behavior.

Document the real state

Good documentation should let another technician understand what exists, what changed, how it was validated, and how to recover.
